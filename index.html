1. JavaScript คืออะไร

ความหมาย

JavaScript (JS) คือ ภาษาคอมพิวเตอร์ในรูปแบบสคริปต์ (Scripting Language) ที่ถูกพัฒนาขึ้นมาเพื่อทำให้หน้าเว็บไซต์มีความเป็น Dynamic หรือมีการเคลื่อนไหว สามารถตอบสนองกับผู้ใช้งานได้แบบเรียลไทม์ ซึ่งแตกต่างจากภาษาโครงสร้างพื้นฐานอย่าง HTML และสไตล์ชีทอย่าง CSS ที่ทำหน้าที่เพียงจัดการโครงสร้างและรูปร่างหน้าตาที่หยุดนิ่งอยู่กับที่ (Static)

ในปัจจุบัน ขอบเขตของภาษา JavaScript ไม่ได้จำกัดอยู่เฉพาะการทำงานภายในเว็บเบราว์เซอร์เท่านั้น แต่ยังแพร่หลายไปยังการพัฒนาแอปพลิเคชันฝั่งเซิร์ฟเวอร์ (Node.js), โมบายแอปพลิเคชัน ตลอดจนวงการเกมมิ่งและ AI อีกด้วย

ประวัติความเป็นมา

จุดเริ่มต้น (1995): ภาษา JavaScript ถูกสร้างขึ้นเป็นครั้งแรกโดย Brendan Eich ในปี ค.ศ. 1995 ระหว่างที่ปฏิบัติงานให้กับบริษัท Netscape Communications ในระยะแรกเริ่มพัฒนาโค้ดถูกเรียกขานภายใต้ชื่อโครงการว่า Mocha ก่อนปรับปรุงมาเป็น LiveScript และในท้ายที่สุดได้ตกลงใจเลือกใช้ชื่อ JavaScript เพื่อร่วมสร้างกระแสความสนใจและเกาะความนิยมของภาษา Java ที่โด่งดังมากในยุคนั้น (แม้ว่าในความเป็นจริงแล้ว ทั้งสองภาษาจะไม่มีโครงสร้างหรือความเกี่ยวข้องกันเลยก็ตาม)

มาตรฐาน ECMAScript: เพื่อขจัดความสับสนและวางรากฐานของอุตสาหกรรมไอทีให้มีมาตรฐานเดียวกัน องค์กร ECMA International จึงเข้ามาทำหน้าที่ดูแลและกำหนดมาตรฐานกลางตั้งแต่ปี ค.ศ. 1997 รู้จักกันในนาม ECMAScript (ES) โดยหมุดหมายสำคัญที่สุดคือการเปิดตัวมาตรฐาน ES6 (ES2015) ที่นำพาฟีเจอร์อันทันสมัย ยืดหยุ่น และทรงพลังมาสู่ระบบนิเวศน์จาวาสคริปต์จนถึงปัจจุบัน

ความสำคัญของ JavaScript

  • หัวใจหลักของ Web Applications ยุคปัจจุบัน: แพลตฟอร์มระดับโลก เช่น Facebook, YouTube หรือ Google ขับเคลื่อนกลไกหลังบ้านและหน้าบ้านด้วย JavaScript เพื่อให้ผู้ใช้งานส่งคำสั่ง กดไลก์ คอมเมนต์ หรือดึงข้อมูลใหม่มาแสดงผลได้ทันทีโดยไม่ต้อง Refresh หน้าเว็บใหม่ทั้งหมด

2. การทำงานของ JavaScript

Client-side Computing

การทำงานของสคริปต์ JavaScript ส่วนใหญ่จะทำหน้าที่ในลักษณะ Client-side หมายถึง ชุดโค้ดคำสั่งจะถูกส่งพ่วงและดาวน์โหลดจากเว็บเซิร์ฟเวอร์มาสู่เครื่องคอมพิวเตอร์หรือสมาร์ทโฟนของผู้ใช้งานโดยตรง และเริ่มต้น **ประมวลผลภายในอุปกรณ์ปลายทาง (Client)** ด้วยเว็บเบราว์เซอร์ วิธีการดังกล่าวมีข้อดีอย่างยิ่งในการช่วยลดภาระการประมวลผลและการใช้ทรัพยากรบนเซิร์ฟเวอร์หลัก ส่งผลให้หน้าเว็บประมวลผลผลลัพธ์ได้อย่างรวดเร็วและไม่หน่วง

การทำงานร่วมกับ HTML และ CSS

เพื่อให้เกิดความเข้าใจที่ชัดเจน นักพัฒนาเว็บมักนิยมเปรียบเทียบการสร้างเว็บเพจเสมือนการปลูกสร้างอาคารบ้านเรือน:

  • HTML: เปรียบเสมือน โครงสร้างดิบของบ้าน (เสาเข็ม คานอิฐ ผนังคอนกรีตปูนเปลือยที่ยังไม่มีสีสัน)
  • CSS: เปรียบเสมือน สถาปัตยกรรมการตกแต่ง (การลงสีห้อง ติดวอลเปเปอร์ จัดวางตู้เตียงผ้าม่านให้สวยงาม)
  • JavaScript: เปรียบเสมือน ระบบกลไกและระบบอัจฉริยะในบ้าน (ระบบเซนเซอร์เปิดไฟอัตโนมัติเมื่อก้าวเดิน, ประตูเลื่อนไฟฟ้า, ระบบลิฟต์โดยสาร)

ภาษา JavaScript จะผสานการทำงานเพื่อควบคุมและปรับเปลี่ยนเนื้อหา HTML และสไตล์ CSS ผ่านกลไกส่วนกลางที่เบราว์เซอร์สร้างขึ้นที่เรียกว่า DOM (Document Object Model) ตัวอย่างเช่น การเขียนสคริปต์สั่งให้ข้อความเกิดการเปลี่ยนสีเมื่อคลิก หรือทำให้ปุ่มแสดงหน้าต่างแจ้งเตือน (Modal) เด้งแสดงผลขึ้นมา

การประมวลผลผ่าน Web Browser Engine

ภายในโปรแกรมเว็บเบราว์เซอร์ชั้นนำ (เช่น Google Chrome, Microsoft Edge, Safari) จะประกอบด้วยเทคโนโลยีหัวใจหลักที่เรียกว่า JavaScript Engine (ตัวอย่างเช่น Engine V8 ของค่าย Chrome/Edge หรือ JavaScriptCore ของค่าย Apple Safari) ทำหน้าที่คอยอ่าน Source Code ที่เราเขียนขึ้น จากนั้นนำไปตีความและแปลงสภาพเป็นภาษาเครื่อง (Machine Code) ที่หน่วยประมวลผลฮาร์ดแวร์เข้าใจโดยตรงอย่างมีประสิทธิภาพ ผ่านเทคนิคการคอมไพล์แบบ Just-In-Time (JIT) Compilation

3. โครงสร้างพื้นฐานของ JavaScript

การเชื่อมไฟล์ JavaScript เข้าสู่โปรเจกต์

ในการพัฒนาเว็บเพจ เราสามารถนำเข้าชุดคำสั่งสคริปต์ JavaScript เข้าไปทำงานร่วมกับเอกสารเว็บได้ 2 แนวทางหลัก:

1. การเขียนโค้ดภายในตัวเอกสาร (Internal JavaScript): เป็นการฝังชุดคำสั่งไว้ภายใต้แท็ก <script> ในหน้าไฟล์ HTML นั้นๆ โดยตรง เหมาะสมกับสคริปต์ขนาดสั้นที่ไม่ซับซ้อน

<!DOCTYPE html>
<html>
<body>
    <h1>หน้าเว็บทดสอบภายใน</h1>
    
    <script>
        console.log("ยินดีต้อนรับสู่ Internal Script!");
    </script>
</body>
</html>

2. การเขียนแยกไฟล์ภายนอก (External JavaScript): เป็นรูปแบบมาตรฐานสากลที่แนะนำ โดยแยกโค้ดสคริปต์ไปเก็บไว้ในไฟล์เฉพาะตัวที่มีนามสกุล .js แล้วทำการเรียกเชื่อมโยงเข้ามาใน HTML ผ่านแอตทริบิวต์ src ช่วยให้การจัดการซอร์สโค้ดเป็นระบบและนำกลับมาใช้ซ้ำในหน้าอื่นๆ ได้ง่าย

<!-- โครงสร้างภายในเอกสาร index.html -->
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <script src="main-app.js" defer></script>
</head>
<body>
    <h1>โครงสร้างหน้าเว็บหลัก</h1>
</body>
</html>

การเขียนสเตตเมนต์คำสั่ง (Statements)

คำสั่งหรือสเตตเมนต์ใน JavaScript คือข้อความคำสั่งที่เขียนระบุขึ้นเพื่อให้คอมพิวเตอร์ประมวลผลพฤติกรรมบางประการ โดยตามหลักไวยากรณ์สากล แต่ละชุดคำสั่งมักจะจบประโยคด้วยเครื่องหมาย Semicolon ( ; ) (แม้ว่าในปัจจุบัน ตัวคอมไพเลอร์ยุคใหม่จะสามารถอนุโลมให้ละเว้นการใส่ได้ แต่การเขียนปิดท้ายด้วยเซมิโคลอนจะช่วยป้องกันความสับสนและบั๊กที่อาจเกิดขึ้นจากการตีความโค้ดผิดพลาด)

ตัวอย่างโครงสร้างคำสั่งพวกรหัสพื้นฐาน:

  • การประกาศสร้างตัวแปรเพื่อเก็บข้อมูล: let developerName = "ปริญญา";
  • การส่งค่าออกไปแสดงผลในหน้าต่างตรวจสอบ: console.log("ชื่อผู้พัฒนา: " + developerName);
  • การประมวลผลเชิงคณิตศาสตร์: let summaryResult = 50 + 150;

การคอมเมนต์อธิบายรหัสต้นฉบับ (Comment)

Comment คือคำอธิบายหรือบันทึกข้อความที่เราพิมพ์แทรกไว้ในหน้าต่างซอร์สโค้ดเพื่อชี้แจงรายละเอียด ฟังก์ชัน หรือวัตถุประสงค์ของโปรแกรมให้ตัวเราเองหรือโปรแกรมเมอร์ในทีมได้รับทราบ โดยตัวเบราว์เซอร์และ JavaScript Engine จะทำการ **ข้ามการประมวลผลคำสั่งในส่วนนี้ไปโดยสิ้นเชิง** มีรูปแบบหลัก 2 ประเภทคือ:

1. คอมเมนต์ประเภทบรรทัดเดียว (Single-line Comment): เริ่มต้นด้วยเครื่องหมายสแลชคู่ //

// บันทึก: ประกาศตัวแปรคงที่เพื่อกำหนดค่าอัตราภาษีมูลค่าเพิ่ม
const vatRate = 0.07;

2. คอมเมนต์ประเภทครอบคลุมหลายบรรทัด (Multi-line Comment): เริ่มต้นด้วยเครื่องหมาย /* และปิดท้ายกลุ่มข้อความด้วย */

/*
  โปรแกรมฟังก์ชันย่อยชุดนี้
  พัฒนาขึ้นโดย ปริญญา พันธุ์วงศา
  วัตถุประสงค์: เพื่อคำนวณราคาสินค้าสุทธิหลังจากหักส่วนลดสะสม
*/
let netPrice = totalPrice - rewardDiscount;
